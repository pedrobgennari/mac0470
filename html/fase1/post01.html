Desenvolvimento do Kernel Linux

> Criando um Ambiente de Testes 🔗



Introdução

Para qualquer software que queremos desenvolver, é uma boa ideia configurar um ambiente que nos permita testar nossas modificações de forma segura e eficiente. O desenvolvimento do Kernel Linux não é diferente; na verdade, devido à sua complexidade, é ainda mais importante o uso de um ambiente de desenvolvimento isolado. Assim, começaremos o ciclo de desenvolvimento do kernel linux configurando uma máquina virtual (VM). Esse tutorial acabou gerando algumas dificuladades. Além de eu não ter muita experiência com Linux, mexer com o Kernel envolve diversos conceitos não muito triviais.


Preparação

Após instalar as depedências, criamos o diretório principal do nosso projeto, chamado lk_dev. Nele, criamos o script activate.sh. Aqui já apareceu o primeiro problema: como foi preciso alterar as permissões do diretório principal, é necessário sair e entrar da sua conta para que as permissões atualizem corretamente (o que, até então, eu não sabia). Eventualmente (depois de mais tempo do que eu gostaria de adimitir), consegui criar o script. Neste tutorial, utilzamos as ferramentas QEMU e libvirt para rodar e a VM. Com a primeira delas, criamos a função launch_vm_qemu que inicializa uma máquina virtual.


Inicializando a VM

Agora, instalamos a imagem de disco do que usaremos com a VM. Estamos interessados no kernel e no initrd da imagem. Executando launch_vm_qemu com o caminho apropriado do kernel e do initrd, a nossa primeira máquina virtual é criada. Agora, o tutorial ensina a user o libvirt. Aqui, outra dificuldade fica clara: são muitos comandos e conceitos que temos que aprender de uma vez, o que acaba tornando o tutorial bastante sobrecarregado. Criamos uma função create_vm_virsh em nosso script, para que possamos subir a VM rápidamente.
