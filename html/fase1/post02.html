Desenvolvimento do Kernel Linux

> Compilando o Kernel Linux 🔗



Introdução

Agora que conseguimos inicializar uma VM, vamos tentar a compilar um kernel customizado e usá-lo na VM. A principal ferramenta aqui é o kw (ou kworkflow). Da mesma forma que no tutorial anterior, alguns problemas surgiram ao longo deste.


Árvore do Kernel Linux e kw

Queremos fazer modificações no kernel linux; logo, precisamos clonar um repositório que contém o código fonte do linux. Para essa série de tutoriais, clonamos o subsistema IIO (Industrial I/O). Especificamente, clonamos a branch testing com um limite do histótico de commits do git, a fim de reduzir o tamanho. Agora, entramos no diretório que contém a árvore e inicializamos e configuramos o kw. Também usamos o kw e o make para configurar a compilação. Por fim, escolhemos um nome para o nosso kernel customizado. Escolhi chamá-lo de "batata".


Compilando o Kernel

Finalmente, depois de mais algumas configurações, executamos kw build para compilar o kernel. O tempo de compilação foi de aproximadamente 13 minutos e tudo pareceu ocorrer bem. Agora, o tutorial atualiza as funções do activate.sh para que utilizem o kernel que compilamos. Porém, nessa parte, me deparei com um erro bem chato de descobrir. O tutorial pede para utilizar "vda" no nome da partição. Porém, sempre que eu tentava abrir a VM com create_vm_virsh, alguma falha acontecia. Descobri depois de algum tempo testando soluções que, na função create_vm_virsh, precisavamos utilizar "sda" no lugar de "vda". Com essa alteração feita, a VM iniciou. Abrindo a VM e executando uname --kernel-release, temos como resultado: 7.0.0-rc7-batata+
