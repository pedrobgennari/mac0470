Desenvolvimento do Kernel Linux

> Enviando Patches com email USP 🔗



Introdução

Contribuições para o Kernel Linux são feitas via email. Algumas ferramentas, como o git send-email ou o próprio kw, podem ser utilizadas para facilitar o processo. Porém, como queremos fazer as contribuições com nossos emails USP, precisamos fazer algumas configurações adicionais.


OAuth 2.0

O tutorial apresenta duas opções para utilizar o protocolo OAuth: git credential helper, ou Email Proxy. Até o momento, porém, o tutorial apenas explica a segunda opção, então optei por ela. Para continuar com essa opção, era necessário instalar e configurar docker e docker-compose. Essa foi a parte mais difícil desse tutorial; não porque foi complicada, mas porque o tutorial em si pulou essa parte. Tendo ambos instalados, não houve nenhum problema com o restante do tutorial e, em alguns minutos, consegui enviar um email com kw send-patch.
